Apple has always understood something that the rest of the industry forgets:
continuity matters. When Apple moved from 68k to PowerPC, then to Intel, then
to Apple Silicon, they did not merely change CPUs. They built bridges.[^continuity]

Those bridges were technical, but they were also cultural. They told people
that the work already done inside a machine still mattered. A saved document,
a familiar menu, a tiny utility that fit the hand perfectly: none of these
things were disposable just because the processor changed underneath them.

Systemless begins from the same instinct. Old software is not only nostalgia.
It is a preserved interface between a person and a machine, and interfaces
carry values. They reveal what the software thought a user was capable of, what
it hid, what it trusted, and what it made easy.

To preserve a program properly, it is not enough to make it execute. The
surrounding world has to be plausible. The file system, screen, memory model,
fonts, window chrome, timings, and failures are all part of the piece.

That is why old software can feel like art. It is bounded, opinionated, and
strangely complete. A small program from another era can make a modern web page
feel weightless by comparison, because the older program knew exactly what
world it was trying to inhabit.

[^continuity]: Continuity here is not sameness. It is the work of letting one
machine understand enough of another machine that human work can cross the gap.
